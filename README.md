პოსტალონი Unsplash API-დან შემთხვევითი სურათის მოტანისა და Firebase Realtime Database-ში პოსტად გამოქვეყნების Android აპლიკაციაა. მომხმარებლებს შეუძლიათ ეს პოსტები მოიწონონ ან არ მოიწონონ. პროექტი Flutter-ით დაწერილი ორიგინალური აპლიკაციის Kotlin-სა და Jetpack Compose-ით გადაწერილი ვერსიაა.
არქიტექტურულად გამოყენებულია MVVM პატერნი. მონაცემთა შრეში არსებობს Firebase-სა და Unsplash-თან კომუნიკაციის სერვის კლასები. ViewModel შრე შეიცავს ბიზნეს ლოგიკას და UI-ს StateFlow-ის მეშვეობით მიაწოდებს მონაცემებს. View შრე კი მთლიანად Jetpack Compose-ით არის შექმნილი.
აპლიკაციის გახსნისას Firebase Anonymous Authentication-ის მეშვეობით ავტომატური ანონიმური შესვლა ხდება, მომხმარებელს არანაირი რეგისტრაციის მონაცემი არ ეთხოვება. Explore ეკრანზე Firebase Realtime Database-დან ბოლო 10 პოსტი რეალურ დროში ისმენება და სია ყოველი ცვლილებისას ავტომატურად განახლდება. Create ეკრანზე Unsplash API-დან შემთხვევითი სურათი იტვირთება, მომხმარებელს გადახედვის საშუალება ეძლევა და დადასტურების შემდეგ Firebase-ში პოსტად ინახება. თითოეულ მომხმარებელს შეუძლია პოსტზე მხოლოდ ერთი ხმა მისცეს, შეცვალოს ან გააუქმოს იგი. Like და dislike-ის მრიცხველები Firebase transaction მექანიზმით ატომურად განახლდება, ანუ თუ ერთდროულად რამდენიმე მომხმარებელი ხმას მისცემს, მრიცხველები მაინც თანმიმდევრული დარჩება. ღილაკზე დაჭერისას ეკრანი Firebase-ის პასუხის მოლოდინის გარეშე მაშინვე განახლდება, რეალური შედეგი კი მოსვლისთანავე სინქრონიზდება — ამას optimistic UI ეწოდება.
გამოყენებული ძირითადი ტექნოლოგიებია: Kotlin გამოყენებულია როგორც მთავარი ენა. Jetpack Compose-ით შეიქმნა UI. Firebase Authentication ანონიმურ შესვლას, Firebase Realtime Database კი პოსტის მონაცემებს მართავს. Unsplash API გამოიყენება შემთხვევითი სურათების წყაროდ. Coil ბიბლიოთეკით სურათები ქსელიდან იტვირთება. Kotlin Coroutines და Flow-ით მართულია ასინქრონული ოპერაციები და რეალურ დროში მონაცემთა ნაკადი.
დაყენებისთვის ჯერ Firebase Console-დან Android აპლიკაცია უნდა შეიქმნას და google-services.json ფაილი პროექტს დაემატოს. შემდეგ Unsplash Developer გვერდიდან მიღებული Access Key Credentials ფაილში ჩაიწერება. Firebase-ის მხრიდან Anonymous Authentication უნდა გააქტიურდეს და Realtime Database-ის წესები ისე უნდა დაყენდეს, რომ შესული მომხმარებლებს ჩაწერა შეეძლოთ.
მონაცემების მხრივ თითოეული პოსტი შეიცავს მომხმარებლის იდენტიფიკატორს, სურათის ბმულს, შექმნის დროს, მოწონებების რაოდენობას, არმოწონებების რაოდენობას და თითოეული მომხმარებლის ხმის შემცველ map-ს. ამ map-ში 1 მნიშვნელობა მოწონებას, 2 კი არმოწონებას წარმოადგენს.
